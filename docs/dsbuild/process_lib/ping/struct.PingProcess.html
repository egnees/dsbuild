Expand description

PingProcess sends consecutive ping messages to the partner process with specified delay. It waits for the consecutive pong responses, while the last pong response is not received.

Fields§

§last_pong: u32

Specifies number of last received pong, initially equals to zero.

§delay: f64

Specifies delay between consecutive pings.

§partner: String

Specifies name of partner process, which must send pongs in response to pings. For example, it can be PongProcess.

§need_cnt: u32

The number of pongs, which are needed to be received.

§is_started: bool

Indicates whether the process is started.

§is_stopped: bool

Indicates whether the process is stopped.

fn on_message( &mut self, msg: Message, from: String, ctx: &mut dyn Context ) -> Result<(), String>

Called when message is received. If received message tip is equal to PONG_TIP, then process tripes to extract pong number from the message. If the number if equals to the expected pong number, then expected pong number is incremented. If after that last received pong number is equal to the expected number of pongs, then process is stopped.
