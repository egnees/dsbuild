Struct Context

pub struct Context { /* private fields */ }
Expand description

Handle which allows process to interact with external environment.

If implemented system is running within the simulation, then context allows to interact with simulation. Else, system is running in the real environment and context will interact with it. Using context process can send network messages to other processes, set timers, manipulate with files and sent local messages to user. Context is passed to the process by the system on every request to handle the external world event.

For more details refer to Process documentation.

pub async fn send_with_tag( &self, msg: Message, tag: Tag, to: Address, timeout: f64, ) -> SendResult<()>

Allows to reliable send network message with specified tag.

On awaiting method will be blocked until the destination will not receive message and send acknowledgment. If acknowledgment was not received for timeout seconds, method will return with Timeout.

Note that receiver will not be explicitly notified about message was tagged, so it is on user to additionaly pass tag within the message if it is needed.

pub async fn send_recv_with_tag( &self, msg: Message, tag: Tag, to: Address, timeout: f64, ) -> SendResult<Message>

Allows to reliable send network message with specified tag and wait for response.

On awaiting method will be blocked until sender process will receive message with the same tag. On success, the received message will be returned.

If message with provided tag will not be received in timeout seconds, method will return with Timeout.

See send_with_tag documentation for more details.
